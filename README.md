## 🧠 Project Overview

EduMentor AI delivers five core features:

| Feature | Description |
|---------|-------------|
| **Course Q&A Chatbot** | RAG-based Q&A grounded in your course documents, with source citations |
| **Exercise Generator** | LLM generates structured coding exercises with hints and solutions |
| **Auto-Grader** | AI evaluates student solutions with structured feedback |
| **Dropout Risk Predictor** | Supervised ML model predicts student dropout risk from activity data |
| **Smart Agent** | When a student is flagged High Risk, the agent triggers recommendations + a remedial quiz |

```
User (React Frontend)
        │
        │ REST API
        ▼
┌─────────────────────────────────┐
│         Flask Backend           │
│                                 │

│       │               │         │
│  ┌────▼─────┐  ┌──────▼──────┐  │
│  │ ChromaDB │  │  ML Model   │  │
│  │Retriever │  │(.joblib)    │  │
│  └────┬─────┘  └──────┬──────┘  │
│       │               │         │
│  ┌────▼─────────────▼──────┐   │
│  │      LLM (Anthropic /   │   │
│  │         OpenAI)         │   │
│  │  Answer + Exercise +    │   │
│  │  Agent Recommendations  │   │
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

**Data Flow — RAG Chat:**
1. User asks question
2. Query embedded with `all-MiniLM-L6-v2`
3. Top-5 relevant chunks retrieved from ChromaDB
4. Prompt = System + Context + Question sent to LLM
5. Answer + sources returned to user

**Data Flow — Risk Prediction:**
1. Student CSV uploaded
2. Features encoded + normalized
3. `risk_model.joblib` predicts probability
4. If High Risk → Agent generates chapter recommendations + quiz via LLM

## 🚀 Quick Start

### Prerequisites
- Python 3.10+
- Node.js 18+
- An API key from [Anthropic](https://console.anthropic.com) or [OpenAI](https://platform.openai.com)

---

### 1. Clone & Setup

```bash
cd EduMentor_Project
```

### 2. Backend Setup

```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env — add your ANTHROPIC_API_KEY or OPENAI_API_KEY
```

### 3. Train the ML Model

```bash
# From project root, open Jupyter
cd ..
jupyter notebook notebooks/preprocessing.ipynb
# Run all cells → generates data/clean_data.csv

jupyter notebook notebooks/train_model.ipynb
# Run all cells → generates backend/models/risk_model.joblib
```

### 4. Ingest Documents

```bash
cd backend
python -c "from rag.ingest import ingest_documents; ingest_documents('../knowledge_base')"
```

Or via API after starting the server:
```bash
curl -X POST http://localhost:5000/api/ingest
```

### 5. Start Backend

```bash
cd backend
python app.py
# Server running at http://localhost:5000
```

### 6. Frontend Setup & Start

```bash
cd frontend
npm install
npm run dev
# App running at http://localhost:5173
```

## 📦 Module Guide

### Module 1 — Data Preprocessing (`notebooks/preprocessing.ipynb`)

Run this first. It:
- Loads `data/raw_dataset.csv`
- Performs EDA (charts saved to `data/`)
- Handles missing values (median/mode fill)
- Encodes: gender → {0,1}, education level → {0,1,2,3}, disability → {0,1}
- Creates target: `at_risk` = 1 if Fail or Withdrawn
- Normalizes numeric columns with MinMaxScaler
- Saves `data/clean_data.csv`

### Module 2 — ML Model (`notebooks/train_model.ipynb`)

Run after preprocessing. It:
- Trains Logistic Regression (with StandardScaler) and Random Forest
- Compares with F1-Macro, ROC-AUC, Confusion Matrix
- Runs 5-fold cross-validation
- Saves best model to `backend/models/risk_model.joblib`

### Module 3 — RAG System

**Ingestion** (`backend/rag/ingest.py`):
- Loads PDF, MD, TXT from `knowledge_base/`
- Chunks at 700 characters with 100-character overlap
- Generates embeddings with `all-MiniLM-L6-v2`
- Stores in ChromaDB (persistent)

**Retrieval** (`backend/rag/retriever.py`):
- Embeds user query
- Retrieves top-5 chunks by cosine similarity
- Filters by relevance threshold (distance < 0.7)
- Returns chunks + unique source filenames

### Module 4 — Exercise Generator

POST `/api/generate-exercise` with `{"topic": "...", "difficulty": "Beginner|Intermediate|Advanced"}`

Returns structured exercise with: Problem → Requirements → Hints → Solution (hidden by default)

POST `/api/grade` evaluates student solutions with: Grade (A-F) → What Works → Improvements → Feedback

### Module 5 — Analytics & Agent

POST `/api/predict-risk` accepts CSV upload.

Risk classification:
- **High Risk**: score ≥ 0.65
- **Medium Risk**: 0.35 ≤ score < 0.65
- **Low Risk**: score < 0.35

High Risk triggers the agent (`backend/ml/agent.py`):
- Generates personalized chapter recommendations (JSON)
- Creates a 3-question remedial quiz (JSON)

## 🏋️ Training the ML Model

1. Open `notebooks/preprocessing.ipynb` and run all cells
2. Open `notebooks/train_model.ipynb` and run all cells
3. Confirm `backend/models/risk_model.joblib` was created
4. Restart Flask backend to pick up the new model

**Expected performance** (on synthetic dataset):
- F1-Macro: ~0.85+
- ROC-AUC: ~0.90+

For better performance, replace `data/raw_dataset.csv` with the OULAD dataset from https://analyse.kmi.open.ac.uk/open_dataset

## 🐛 Troubleshooting

**"No collection found"**: Run ingestion first: `POST /api/ingest`

**"Model not found"**: Run both notebooks to generate `risk_model.joblib`

**CORS errors**: Ensure `CORS_ORIGINS` in `.env` includes your frontend URL

**LLM not responding**: Check your API key in `.env`. The app has fallbacks for all LLM calls.

**ChromaDB errors**: Delete the `./chroma_db` folder and re-ingest
